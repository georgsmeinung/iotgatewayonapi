IoT Gateway on a Raspberry Pi

Gateway de IoT en una Raspberry Pi

Welcome to the complete documentation hub. This project analyzes and demonstrates the protocol stack needed to implement a local, secure, and responsive domotic central (IoT Gateway) utilizing a Raspberry Pi Zero W and ESP8266 Microcontrollers. Bienvenido al centro de documentación completo. Este proyecto analiza y demuestra la pila de protocolos necesarios para implementar una central domótica local, segura y responsiva (Gateway IoT) utilizando una Raspberry Pi Zero W y microcontroladores ESP8266.

Trabajo Final Integrador: Domotic Control Central

Trabajo Final Integrador: Central de Control Domótico

Academic thesis submitted by Jorge Enrique Nicolau describing a home automation central focused on lighting control and presence simulation. Tesis académica presentada por Jorge Enrique Nicolau que describe una central de automatización del hogar enfocada en el control de iluminación y simulación de presencia.

1. Objective of the SolutionObjetivo de la Solución

The main objective is to describe a domestic electronic control system, its hardware/software components, communication protocols, and visual representation. The platform features both mobile and web interfaces allowing users to monitor ambient parameters, switch actuators manually, set automated routines, and execute presence simulation to improve home security. El objetivo principal es describir una solución electrónica de uso doméstico, sus componentes hardware y software, protocolos de comunicación e interfaz visual. La plataforma cuenta con interfaces móviles y web que permiten al usuario monitorear parámetros ambientales, operar actuadores manualmente, programar rutinas automatizadas y ejecutar una simulación de presencia para mejorar la seguridad del hogar.

2. Functional DescriptionDescripción Funcional

Wired and wireless transceiver nodes communicate with a central coordinator. Wired nodes rely on Arduino Uno coupled with an Ethernet shield, connected via physical LAN wiring. Wireless nodes utilize NodeMCU boards, communicating over local Wi-Fi. Nodos transceptores cableados e inalámbricos se comunican con un coordinador central. Los nodos cableados utilizan placas Arduino Uno junto con un protector Ethernet, conectados por cable LAN. Los nodos inalámbricos utilizan placas NodeMCU, comunicándose a través de la red Wi-Fi local.

Actuators:Actuadores: Relays read status topics regularly and toggle active loads (e.g. lamps, heaters). High voltage isolations are done via optocouplers. Los relés leen los tópicos de estado con regularidad y alternan cargas activas (ej. lámparas, calentadores). Las aislaciones de alta tensión se realizan mediante optoacopladores.

Sensors:Sensors: Photoelectric LDR modules analyze ambient light. Temperature and humidity values are read from DHT22 sensors and published to the local broker. Los módulos LDR fotoeléctricos analizan la luz ambiental. Los valores de temperatura y humedad se leen desde sensores DHT22 y se publican en el broker local.

3. Gateway and Rule Engine IntegrationIntegración de Gateway y Motor de Reglas

The gateway consists of a Raspberry Pi Zero W hosting a Mosquitto MQTT broker and a Node-RED engine. Node-RED maps topics dynamically and links sensors directly to actuators. For instance, when humidity readings drop below a threshold, the engine publishes an 'on' message to the humidifier topic automatically. El gateway consta de una Raspberry Pi Zero W que aloja un broker Mosquitto MQTT y un motor Node-RED. Node-RED mapea tópicos dinámicamente y vincula sensores directamente a actuadores. Por ejemplo, cuando las lecturas de humedad caen por debajo de un umbral, el motor publica automáticamente un mensaje 'on' en el tópico del humidificador.

Why MQTT over HTTP?¿Por qué MQTT sobre HTTP?

Compared to HTTP, MQTT is vastly superior for resource-constrained edge computing. It stays online in a persistent TCP connection, uses very small packets (less overhead), and saves significant power for battery-operated devices. It represents a real-time data channel. En comparación con HTTP, MQTT es muy superior para la computación en el borde con recursos limitados. Permanece en línea en una conexión TCP persistente, utiliza paquetes muy pequeños (menos sobrecarga) y ahorra energía significativa para dispositivos alimentados por batería. Representa un canal de datos en tiempo real.

/**
 * ESP8266 MCU routine for DHT11 reading and MQTT writing
 */
#include <SDHT.h>
#include <PubSubClient.h>
#include <ESP8266WiFi.h>

const char* IOTGATEWAY = "nn.nn.nn.nn"; // Gateway IP
const char* WIFISSID = "********"; 
const char* WIFIPASS = "********"; 

const char* TOPICTEMP = "/casa/temperatura";
const char* TOPICHUMI = "/casa/humedad";
const char* TOPICCTRL = "/casa/humidificador";
const char* TOPICTEST = "hello_topic";

#define DHTPIN 14     
#define LEDPIN 13     
#define DHTTYPE DHT11  
#define REPORT_INTERVAL 1
#define SERIALBPS 9600 

SDHT dht;
String clientName;
WiFiClient wifiClient;

void callback(char* topic, byte* payload, unsigned int length) {
    Serial.print(">> Message arrived: ");
    String message;
    for (int i = 0; i < length; i++) {
        message = message + (char)payload[i];
    }
    Serial.print(message);
    if (message == "on") digitalWrite(LEDPIN, HIGH);
    if (message == "off") digitalWrite(LEDPIN, LOW);
}

PubSubClient client(IOTGATEWAY, 1883, callback, wifiClient);

void setup() {
    Serial.begin(SERIALBPS);
    pinMode(LEDPIN, OUTPUT);
    WiFi.begin(WIFISSID, WIFIPASS);
    while (WiFi.status() != WL_CONNECTED) {
        delay(500);
    }
    clientName += "esp8266-";
    if (client.connect((char*)clientName.c_str())) {
        client.subscribe(TOPICCTRL);
    }
}

void loop() {
    client.loop();
    if (dht.read(DHTTYPE, DHTPIN)) {
        double temp = dht.celsius / 10.0;
        double humi = dht.humidity / 10.0;
        client.publish(TOPICHUMI, String(humi, 1).c_str());
        client.publish(TOPICTEMP, String(temp, 1).c_str());
    }
    delay(REPORT_INTERVAL * 1000);
}
_01.temp_sensor_01.ino (DHT22 Thesis Code) 
// Rutina MCU para ESP8266, lectura DHT22 y escritura con MQTT

#include "DHT.h"
#include <PubSubClient.h>
#include <ESP8266WiFi.h>

const char* ssid = "********"; 
const char* password = "********"; 

char* topicTemp = "/casa/temperatura";
char* topicHum = "/casa/humedad";
char* server = "XX.XX.XX.XX"; // Server MQTT (Raspberry Pi Zero W IP)
char* hellotopic = "hello_topic"; 

#define DHTPIN 17     
#define DHTTYPE DHT22   
#define REPORT_INTERVAL 30 

String clientName;
DHT dht(DHTPIN, DHTTYPE, 15);
WiFiClient wifiClient;
PubSubClient client(server, 1883, callback, wifiClient);

float oldH;
float oldT;

void setup() {
  Serial.begin(38400);
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
  }
  
  clientName += "esp8266-";
  if (client.connect((char*) clientName.c_str())) {
    client.publish(hellotopic, "hello from ESP8266");
  }
  dht.begin();
  oldH = -1;
  oldT = -1;
}

void loop() {
  float h = dht.readHumidity();
  float t = dht.readTemperature();

  if (isnan(h) || isnan(t)) {
    Serial.println("¡Falla al leer del sensor DHT!");
    return;
  }

  if (t != oldT || h != oldH ) {
    sendTemperature(String(h), String(t));
    oldT = t;
    oldH = h;
  }
  
  int cnt = REPORT_INTERVAL;
  while (cnt--) delay(1000);
}

void sendTemperature(String plHum, String plTem) {
  if (client.connected()) {
    client.publish(topicHum, (char*) plHum.c_str());
    client.publish(topicTemp, (char*) plTem.c_str());
  }
}
